# 脱敏故障复盘案例

本模块把仪控实习中可公开表达的观察，转换成可播放的训练案例。它不复现真实生产数据，也不把未验证推断写成事故结论。

## 数据边界

- 只使用脱敏后的症状、处置路径和复测逻辑。
- 不包含厂名、机组号、具体时间、点位、运行参数、截图、人员或内部制度。
- 每一步显式标记为“事实”“推断”或“待确认”。
- “产品机会”是作品集中的产品假设，不是现场改造建议。

## 案例一：温度信号失真

读数明显超出常用量程后，先核对过程合理性，再沿现场仪表、I/O、DCS 和工作票链路排查。确认断连后，在许可条件下修复并复测闭环。

训练重点：信号合理性判断、链路化排查、修复后验证。

## 案例二：重复液位测量故障

同类液位测量异常近期重复出现。污染或凝结可能影响雷达回波是现场推断，尚不能作为确认根因；后续清理、调整或改造效果需要复测证明。

训练重点：重复故障建档、事实与假设分离、根因关闭追踪。

## 案例三：压力表消缺判断

两处相似低读数或不归零表象，通过表计状态与设备状态区分：一处确认失准并更换，另一处反映关联设备未启动时的真实状态。

训练重点：把设备状态、隔离条件和仪表自检一起纳入消缺判断。

## 数据文件

- 可视化案例：`data/public/incidents.json`
- 结构约束：`schemas/incidents.schema.json`
- 运行校验：`npm run validate:data`
