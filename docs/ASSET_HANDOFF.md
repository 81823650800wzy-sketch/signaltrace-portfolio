# 轻量三维资产交接

## 当前边界

ThermalFlow Lab 当前只使用轻量化体块、设备锚点和流程连线，不创建精细 Blender 模型，不进行纹理烘焙，也不复刻现场精确结构。

这样保留三维与实景结合的接口，同时把制作额度集中在流程解释、故障复盘、报告和产品案例上。

## 后续 3D AI 交付要求

专用 3D 创建 AI 生成的每个资产应以 GLB 交付，并附带以下元数据：

| 字段 | 要求 |
|---|---|
| `assetId` | 稳定、可读的资产标识 |
| `nodeId` | 对应 `process-flow.json` 的流程节点 ID |
| `scaleUnit` | 米制，`meter` |
| `origin` | 底部中心点，便于贴合照片地面或画布锚点 |
| `forwardAxis` | 明确模型正前方轴向 |
| `bounds` | 简化碰撞/选中边界 |
| `materialSlots` | 可读材质槽名称，不绑定真实厂家标识 |
| `lod` | `low`、`medium` 或 `high`，Web 默认加载 `low` |
| `license` | 素材来源与可公开使用说明 |

## 实景结合

只有用户确认可公开的现场照片或全景图，才能作为背景或对齐参考。未获批准时使用自制或公开替代素材；不输入内部图纸、DCS 截图、精确坐标、设备位号或未授权照片给外部模型服务。

## 替换方式

前端先按 `nodeId` 绑定占位体块。资产到位后，以同一 `nodeId` 的 GLB 替换，不改变流程数据、案例数据和交互逻辑。
