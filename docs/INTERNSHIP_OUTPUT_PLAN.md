## 六阶段实操

### 阶段 0：事实与边界（现在）

- 已确认岗位为仪控班、周期约一个半月；准确起止日期可在报告定稿时补充。
- 现场照片和机组编号由用户自行选择并补充，进入公开仓库前执行脱敏检查。
- 为日志去重，给每条事实加日期、来源、置信度和隐私等级。
- 建立公开资料登记表，厂级事实至少需要一份政府或企业公开来源。

### 阶段 1：流程最小演示

- 先做 2D 流程图与粒子，不等待完整三维模型。
- 实现系统筛选、播放/暂停、速度控制、节点卡片和引导模式。
- 用 5 名非电力专业同学测试能否讲清四条主链路。

### 阶段 2：三维与实景

- Blender 建立低多边形主厂房、锅炉、汽机、除尘/脱硫、冷却与升压设施。
- 统一米制、原点、命名、材质与 LOD，导出 GLB。
- 使用获准公开的照片或全景图做背景和热点对齐；无授权时使用自制/公开替代素材。

### 阶段 3：故障复盘

- 优先完成温度异常、重复液位故障和压力表消缺三个脱敏案例。
- 所有趋势数据使用合成数据并显著标识；真实阈值仅在公开或获准时使用。
- 加入“已知/未知/下一步证据”而不是强行给出根因。

### 阶段 4：产品验证

- 访谈至少 2 名同学、1 名新员工/实习生和 1 名有经验人员（如允许）。
- 记录任务完成时间、解释准确率、迷失点和最常点击内容。
- 用证据调整优先级，保留版本前后对比。

### 阶段 5：报告与求职包装

- 实习报告按“流程认知 → 仪控实践 → 案例 → 数字化机会 → 反思”组织。
- 产品案例按“问题 → 研究 → 定义 → 方案 → 取舍 → 验证 → 下一步”组织。
- README 首屏放真实演示截图、30 秒 GIF、个人贡献、数据边界和在线地址。

## AI 分工

| 角色 | 固定职责 |
|---|---|
| Codex | 公开资料校验、数据模型、代码、Blender 自动化、测试、性能、Git/GitHub 包装 |
| Claude/DeepSeek | 低成本读取日志、去重分类、提炼 STAR 事件、周报/报告草稿、第二视角需求审查 |
| 用户 | 现场观察、许可范围确认、补问专业人员、拍摄获准素材、判断事实是否准确 |

Claude 每周只做一次批量日志处理，避免每天重复消耗；Codex 读取它写入的短摘要，不共享原始日志和长对话。

## 每周节奏

- 每天：5 分钟记录事实，5 分钟补“证据/结果/数字化机会”。
- 每周：Claude 批量去重与摘要；Codex 更新一条流程、一张学习卡和一个可演示增量。
- 每两周：完成一次用户测试和一次 README/视频更新。
- 实习结束前一周：冻结事实、补齐结果、完成脱敏审查与报告终稿。
