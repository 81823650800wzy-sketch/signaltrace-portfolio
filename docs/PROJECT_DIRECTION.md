# 项目方向

## 决策

项目名称暂定 **ThermalFlow Lab**，定位为“燃煤机组流程训练与仪控故障复盘的交互原型”。

项目源于一次约一个半月的发电厂仪控班实习。开发期默认使用泛化设备名；现场照片和机组编号由实习生本人确认许可后补充。

这比“完整数字孪生电厂”更可信：当前没有实时生产数据、精确 CAD/BIM、设备协议和厂方授权。项目先证明复杂流程抽象、现场问题洞察、交互设计和工程落地能力；以后拿到合法数据时，再扩展为实时状态映射。

## 目标用户

| 用户 | 核心任务 | 当前困难 | 产品价值 |
|---|---|---|---|
| 实习生/能源专业学生 | 理清设备与介质全流程 | 图纸、现场、DCS 和文字知识割裂 | 用同一场景建立系统心智模型 |
| 新入厂运行/仪控人员 | 熟悉设备、信号与消缺路径 | 位置、位号、历史异常分散 | 设备热点和案例回放降低学习成本 |
| 培训人员 | 讲解流程并验证理解 | 课件静态、缺少互动和反馈 | 可复用的引导模式与测验 |
| 招聘评审者 | 判断候选人的产品与工程能力 | 简历描述难验证 | 从事实、需求到实现的完整证据链 |

## 现场证据到需求

| 日志事实（脱敏） | 用户问题 | 产品机会 | MVP 优先级 |
|---|---|---|---|
| 排查压力表时需要人工逐牌寻找 | 资产位置与位号难关联 | 场景热点、搜索、设备关系图 | P0 |
| 某液位雷达三个月处理 6 次 | 重复故障没有可见的根因闭环 | 故障时间线、原因树、措施验证 | P0 |
| 循泵轴承温度长期接近临界值 | 单点报警缺少趋势与工况上下文 | 合成趋势、阈值解释、关联变量 | P1 |
| 四只仪表因锈蚀无法顺利更换 | 预防维护与现场条件未沉淀 | 失效模式卡片、巡检检查项 | P1 |
| 日志 API 返回重复记录 | 数据进入报告前需要治理 | 去重规则、来源与版本标记 | P0 |

## MVP

1. **流程探索器**：六类流程可独立显隐，支持引导播放和自由浏览。
2. **设备学习卡**：功能、输入/输出、关键观测量、上下游和证据来源。
3. **粒子系统**：方向正确；速度/密度只表达相对状态，除非数值有公开依据。
4. **故障复盘**：至少三个脱敏案例，使用“现象-证据-判断-行动-结果-遗留问题”。
5. **产品证据页**：展示访谈/观察、需求排序、取舍、指标和迭代记录。

## 非目标

- 不复刻厂区精确布局，不公开设备坐标、内部位号表或 DCS 画面。
- 不接入生产网，不模拟真实控制，不给出操作指令。
- 不把合成趋势包装成真实运行数据。
- 不在没有标签和足够样本时宣称实现预测性维护。

## 成功指标

- 新用户完成引导后，能正确复述煤、汽水、烟气和电力四条主链路。
- 用户能在 60 秒内从一个异常点追溯到上下游设备和对应案例。
- 每个功能都能追溯到公开资料或脱敏日志事实。
- GitHub 首屏能在 90 秒内说明问题、方案、个人贡献和结果。
- 面试时可用 3 分钟讲清一个现场问题如何转化为产品决策。

## 求职定位

第一目标岗位应聚焦 **工业互联网、能源数字化、IoT 平台、设备运维或企业级 AI 产品经理**。这项实习项目能直接证明现场研究、复杂流程抽象、B2B 工作流、数据治理、三维表达和跨专业沟通，比泛泛写“参与电厂实习”更有辨识度。

面向大厂通用产品岗时，作品集采用“一主两辅”：

1. **ThermalFlow Lab（主项目）**：证明从现场问题到需求、原型、验证和发布的完整产品闭环。
2. **AI Habit DB（辅助项目）**：证明个人数据、AI 洞察和用户增长/留存思维。
3. **AI Agent Deck（辅助项目）**：证明软硬件协同、嵌入式约束和 AI 工作流理解。

三者不要平均分配篇幅。简历和作品集首页重点讲 ThermalFlow Lab，其余项目用于回答“是否只懂电厂”和“能否跨端落地”。消费互联网产品岗仍需额外补充增长、交易或内容场景案例，不能仅凭电厂可视化项目覆盖。

## 技术架构

```text
公开资料 + 脱敏观察 + 合成数据
              |
       版本化 process graph
        /         |          \
   3D 节点/路径  学习卡片   故障状态机
        \         |          /
       React + Three.js 交互应用
              |
      GitHub / 静态站点 / 演示视频
```

Blender 负责设备外形、厂房尺度关系与烘焙贴图；Web 端负责粒子、热点、筛选、时间线和响应式交互。真实场景结合优先采用“获准拍摄的照片/全景图 + 简化 3D 叠加”，不追求未经授权的精确测绘。
