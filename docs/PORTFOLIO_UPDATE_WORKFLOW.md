# 本地工作台到 Android App 的更新流程

本项目的个人展示内容由本地网站工作台统一整理，再以离线内容包导入 Android App。该流程不连接实习日记网站，不上传内容，也不绕过公开边界审核。

## 使用步骤

1. 在本地运行网站后打开 `http://127.0.0.1:5173/#studio`。
2. 在“内容编辑”中更新公开名称、定位、成果或模型索引；草稿只保存在当前浏览器。
3. 在“实时预览”和“可视化资产”中核对表达、完成状态与公开边界。
4. 点击“导出到 App”，得到 `portfolio.json`。
5. 将该文件传到 Android 设备，在 App 的“更新”页选择“导入 portfolio.json 内容包”。
6. 对需要展示的流程模型，在 App 的“流程”页按模型 ID 关联本地 GLB 或其他模型文件。

## 发布前检查

- 不含真实机组号、精确参数、内部截图、人员信息、账号、凭证或未经批准的媒体。
- 将事实、推断和待确认内容分开表述；未核实内容不作为成果结论。
- 模型条目的 `id` 使用稳定的小写英文 ID，后续模型文件与说明围绕同一个 ID 更新。
- 如果内容只适合个人复盘，不应进入工作台或 Android App 内容包。

## 当前模型能力

本地网站负责流程、资产状态和文本说明的可视化模拟；现有流程实验室提供二维粒子流交互。Android App 支持本地模型文件的关联和交给兼容查看器打开。内置 GLB 渲染器是后续增强，不影响现有内容包格式。
