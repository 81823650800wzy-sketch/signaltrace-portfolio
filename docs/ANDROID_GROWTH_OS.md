# SignalTrace Android Growth OS

## 产品结构

SignalTrace 0.7.0不再把作品集包装成网页容器，而是使用五个长期稳定的原生空间：

1. 名片：个人定位、当前信号、作品统计和模块入口。
2. 实习：总览、现场日志、作品、能力图谱四个内部子界面。
3. 实验：氢气纯度仪响应数据、模型指标、原始视频和判定流程。
4. 愿望：公开默认方向与只保存在本机的个人条目、完成状态。
5. 更新：资源包、APK能力包、校验状态、发布说明和恢复工具。

## 视觉语言

- 母题：上升信号穿过工业棱镜，表示把现场噪声转化为成长证据。
- 基色：石墨黑`#0D1114`、纸白`#EEEFE9`、警示黄`#F6B82E`、信号青`#32CDBE`、故障红`#EE5244`。
- 形态：斜切几何、方形标记、窄体标题、克制边线，不使用通用圆角卡片堆叠。
- 动效：页面空间转场、卡片分层进入、导航几何翻转、展开重排、触摸视差和持续信号运动。

## 更新协议

`content/app-update.json`是唯一远程更新清单。客户端执行：

1. 通过HTTPS获取清单。
2. 对比内容版本和APK`versionCode`。
3. 将资源包下载到应用内部暂存区。
4. 对下载字节执行SHA-256校验。
5. 解析JSON并验证必要模块。
6. 通过同文件系统原子移动替换活动资源包。
7. APK能力包下载到私有缓存，校验后交给Android系统安装器。

任何步骤失败都不会删除当前活动内容。Android不允许普通App静默覆盖安装自身，因此APK最后一步始终需要用户确认。

## 后续成长

- 新增实习日志、作品、默认愿望、实验摘要：只发布成长资源包。
- 新增导航空间、系统权限、渲染器或文件能力：发布APK能力包。
- 原始现场视频、个人愿望和完成状态：保持本地，不进入公开仓库。
- 更新发布必须同时更新版本、文件大小、SHA-256和发布说明。
