# 我的能力与工作方法

## 我希望被如何理解

我是一个正在建立工业现场认知的学习者：能够从流程、仪表、控制信号和维护观察中提取问题，再用可核验的方式组织成文档和交互材料。

我不把短期实习包装成独立负责的工程成果，也不把学习原型宣传为已上线产品。我的价值在于保持事实边界的前提下，持续把复杂、分散的信息变得更可理解、更可复盘。

## 已体现的能力

| 能力 | 我的做法 | 可查看证据 |
|---|---|---|
| 系统理解 | 将六类流程和仪控信号闭环拆解为可浏览的节点关系 | 流程图与 `PROCESS_MAP.md` |
| 问题分析 | 不只看仪表读数，同时考虑设备状态、历史记录与维护条件 | 三个脱敏复盘案例 |
| 证据边界 | 明确事实、推断、待确认；公开、脱敏、私有内容分层 | `INCIDENT_CASES.md` 与本日志 |
| 工具协作 | 用 AI 协助摘要与分类，但由本人复核事实、隐私和表述范围 | 协作规则与版本记录 |
| 结构化表达 | 将现场跟学转化为时间线、事实卡、流程图和复盘叙事 | 本仓库文档结构 |

## 我的工作原则

1. 先确认事实，再讨论解释。
2. 先理解系统上下文，再判断单点异常。
3. 先标明不确定性，再提出下一步验证。
4. 使用 AI 提升整理效率，但不让 AI 替代事实判断和责任边界。
5. 将可公开、可复盘的学习成果持续沉淀，而不是追求一次性包装。

## 对企业读者的说明

本仓库适合了解我的学习方法、工程思考和表达能力，不应用于推断任何现场生产配置。若有机会进入更完整的工程或产品场景，我会继续遵循数据权限、专业复核和安全边界，并在真实约束下完成更扎实的验证。
