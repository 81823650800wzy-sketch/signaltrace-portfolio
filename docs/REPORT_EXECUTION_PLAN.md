# 实习报告优先执行计划

## 当前主目标

ThermalFlow Lab 从“产品开发项目”调整为“实习报告与个人能力证据包”。现有流程图和故障复盘只保留为附录截图、方法证明和面试素材，不再新增产品功能，除非它直接补足报告证据。

## 报告要证明的能力

| 能力 | 可验证证据 | 报告呈现方式 |
|---|---|---|
| 复杂系统理解 | 六条流程链路与仪控信号闭环 | 发电流程认知章节 + 流程图 |
| 现场问题拆解 | 温度、液位、压力表三类脱敏案例 | “现象-证据-判断-行动-结果/待确认”案例表 |
| 工程边界意识 | 公开、脱敏、私有三层数据边界 | 方法与数据来源声明 |
| 结构化表达 | 去重后的事实卡与引用登记 | 实习周记/报告证据台账 |
| AI 协作能力 | Claude/DeepSeek 负责低成本摘要，人工复核事实 | 工具与方法附录，不展示原始对话或凭证 |

## 从今天开始的交付顺序

1. 获取一次脱敏、去重的实习日志摘要，只进入私有证据台账。
2. 为每条摘要补齐“观察、动作、结果、待确认、隐私等级”。
3. 先完成报告第 2、3、4 章初稿：背景目标、流程认知、仪控案例。
4. 从事实卡中筛选 3 条 STAR 素材，写入简历与面试备忘。
5. 最后才选择 1-2 张已获准的图片或流程图截图作为附图。

## 私有证据台账字段

台账仅存放在 `D:\PrivateInternship\thermal-flow-lab`，不提交 Git：

| 字段 | 规则 |
|---|---|
| 日期 | 可保留精确日期；公开稿只保留周或阶段 |
| 事件主题 | 使用泛称，不写点位、机组或内部编号 |
| 观察 | 只写可核实的事实 |
| 行动 | 区分本人观察、协助和现场人员处置 |
| 结果 | 无法确认时写“待确认” |
| 来源 | 日志 / 公开资料 / 现场观察 |
| 隐私等级 | 公开 / 脱敏可用 / 仅私有 |
| 报告去向 | 章节与附录位置 |

## 可直接使用的成果表述

以下句式必须在台账中有对应事实后使用：

- “在约一个半月的仪控实习中，我将分散的流程、信号和消缺观察整理为可追溯的证据台账。”
- “我用事实、推断与待确认三类状态记录重复故障，避免把未验证的现场判断当作结论。”
- “我基于公开资料与脱敏观察构建了流程学习原型，用于解释介质路径和仪控信号闭环，而非模拟真实生产控制。”

## 不做的事

- 不把原型继续扩展成完整产品，不以产品功能数量评价实习成果。
- 不用 AI 生成不存在的现场经历、处置结果或量化成绩。
- 不公开原始日志、精确参数、内部系统截图、人员和设备识别信息。
