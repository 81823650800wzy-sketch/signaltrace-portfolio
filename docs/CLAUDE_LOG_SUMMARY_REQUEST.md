# Claude 实习日志摘要请求

## 使用前提

仅在 Claude Code 已正确配置并能访问秒哒实习日志 API 时使用。该连接可以是 Claude 内部记忆的直接 API 调用，不一定表现为本机 MCP 配置。本仓库不保存 API 地址、密钥、账号、原始返回值或任何凭证。

若连接不可用，不要反复重试或安装未知插件；先在用户自己的 Claude 环境中恢复原有连接后，再执行一次以下请求即可。

## 最小调用提示词

```text
请使用已接入的秒哒实习日志 API，只读取指定日期范围的实习记录并完成一次去重。

严格禁止输出或写入：原始日志全文、姓名、账号、厂名、机组号、点位、精确运行参数、截图文字、内部网络/控制信息、凭证。

只返回一份 Markdown 摘要：
1. 数据质量：记录总数、去重后事件数、日期覆盖、重复或缺失说明。
2. 5-8 条脱敏事实卡：日期（可只到周）、主题、观察、本人参与动作/现场动作、结果或待确认、证据等级、适合放入的报告章节。
3. 3 条 STAR 素材：场景、任务、行动、结果、学到的能力。不得夸大个人贡献。
4. 报告仍缺的关键问题。

如果 API 不可用，只说明不可用原因，不猜测、不补写事实。
```

## 摘要落点

Claude 返回的摘要先由本人核对，再保存到私有目录的证据台账。进入公共报告或 GitHub 前，逐条执行脱敏和事实确认。
