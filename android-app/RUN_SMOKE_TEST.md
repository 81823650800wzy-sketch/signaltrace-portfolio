# Android 作品集冒烟测试

使用虚构、脱敏测试内容。不要导入真实机组号、参数、现场截图、人员信息或内部资料。

1. 安装 `app/build/outputs/apk/debug/app-debug.apk`，确认首页、成果、流程、更新四个页面均可打开。
2. 确认首页封面和三条能力证据可离线显示；断开网络后重新打开 App，内容仍可访问。
3. 在“更新”页导入 [portfolio-content.example.json](content-pack/portfolio-content.example.json) 的一个修改副本，确认首页定位、成果数和模型数即时变化。
4. 在“流程”页为一个模型关联虚构的本地文件，确认状态变为“已关联离线模型文件”。
5. 若设备有兼容查看器，点击“打开”并确认系统文件打开流程出现；没有查看器时，应显示明确提示而不是崩溃。
6. 使用“恢复内置作品集”，确认默认内容恢复。

通过标准：页面可启动、内容包可导入且在重启后保持、离线可读、模型文件关联不要求网络。
